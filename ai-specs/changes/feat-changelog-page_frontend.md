# Frontend Implementation Plan: feat-changelog-page — Changelog Page

## Overview

Add a "Novedades" (What's New) page inside the authenticated app that displays release notes fetched from the GitHub Releases API. Each release shows its version tag, publication date, and a human-readable changelog body (markdown generated by Claude AI via the changelog GitHub Action). The page uses `marked` for markdown rendering and follows the project's composable-based architecture with PrimeVue + Tailwind CSS.

## Architecture Context

- **View**: `frontend/src/views/ChangelogPage.vue` — new page
- **Composable**: `frontend/src/composables/useChangelog.ts` — fetches GitHub Releases API
- **Types**: `frontend/src/types/changelog.ts` — TypeScript interfaces for GitHub Release response
- **Router**: Add `/changelog` route in `frontend/src/router/index.ts` (requiresAuth: true)
- **State management**: Local state only (no Pinia store needed — data is read-only, no shared state)
- **New dependency**: `marked` (lightweight markdown parser, ~40KB)

### Step 3: Create Composable

- **File**: `frontend/src/composables/useChangelog.ts`
- **Action**: Composable that fetches releases from `https://api.github.com/repos/nessuarez/abuvi-app/releases`
- **Important**: This does NOT use the project's Axios instance (`api` from `@/utils/api`) because it calls GitHub's public API, not the backend. Use plain `fetch` or a separate axios call without auth interceptors.
- **Pattern**: Follow `useAssociationSettings.ts` pattern (loading, error, data refs)
- **Implementation**:

```typescript
import { ref } from 'vue'
import type { GitHubRelease } from '@/types/changelog'

const GITHUB_RELEASES_URL = 'https://api.github.com/repos/nessuarez/abuvi-app/releases'

export function useChangelog() {
  const releases = ref<GitHubRelease[]>([])
  const loading = ref(false)
  const error = ref<string | null>(null)

  const fetchReleases = async (): Promise<void> => {
    loading.value = true
    error.value = null
    try {
      const response = await fetch(GITHUB_RELEASES_URL, {
        headers: { 'Accept': 'application/vnd.github+json' }
      })
      if (!response.ok) throw new Error(`GitHub API error: ${response.status}`)
      releases.value = await response.json()
    } catch (err: unknown) {
      error.value = 'No se pudieron cargar las novedades'
      console.error('Failed to fetch releases:', err)
    } finally {
      loading.value = false
    }
  }

  return { releases, loading, error, fetchReleases }
}
```

### Step 4: Create ChangelogPage View

- **File**: `frontend/src/views/ChangelogPage.vue`
- **Pattern**: Follow `ProfilePage.vue` structure (`<script setup lang="ts">`, PrimeVue Card, Container, Tailwind)
- **Layout**:
  - Page title: "Novedades" with subtitle
  - List of releases in reverse chronological order (API already returns them this way)
  - Each release rendered as a PrimeVue `Card` with:
    - Header: Tag name (e.g., "v1.1.1") + formatted date
    - Content: Markdown body rendered to HTML via `marked`
    - Link to GitHub Release page
  - Loading state: PrimeVue `Skeleton` components
  - Error state: PrimeVue `Message` component
  - Empty state: informative message
- **Markdown rendering**: Use `marked.parse()` to convert body to HTML, render with `v-html`
- **Styling**: Use Tailwind `prose` classes for markdown content (requires `@tailwindcss/typography` plugin) OR style the markdown output with custom Tailwind utility classes on the container.
- **Important note on Tailwind CSS 4**: The project uses Tailwind CSS 4.1. Check if `@tailwindcss/typography` is compatible. If not, apply manual styles for markdown elements (headings, lists, paragraphs) using Tailwind classes on a wrapper div.

```vue
<script setup lang="ts">
import { onMounted, computed } from 'vue'
import { marked } from 'marked'
import Card from 'primevue/card'
import Skeleton from 'primevue/skeleton'
import Message from 'primevue/message'
import Button from 'primevue/button'
import Container from '@/components/ui/Container.vue'
import { useChangelog } from '@/composables/useChangelog'

const { releases, loading, error, fetchReleases } = useChangelog()

const formatDate = (iso: string) =>
  new Intl.DateTimeFormat('es-ES', {
    day: '2-digit', month: 'long', year: 'numeric'
  }).format(new Date(iso))

const renderMarkdown = (md: string): string => {
  return marked.parse(md) as string
}

onMounted(() => fetchReleases())
</script>

<template>
  <Container>
    <div class="py-8 space-y-6 max-w-4xl">
      <div>
        <h1 class="text-4xl font-bold text-gray-900">Novedades</h1>
        <p class="mt-1 text-sm text-gray-500">Historial de cambios y mejoras de la plataforma</p>
      </div>

      <!-- Loading -->
      <div v-if="loading" class="space-y-4">
        <Card v-for="i in 3" :key="i">
          <template #content>
            <Skeleton width="30%" height="1.5rem" class="mb-2" />
            <Skeleton width="20%" height="0.75rem" class="mb-4" />
            <Skeleton width="100%" height="4rem" />
          </template>
        </Card>
      </div>

      <!-- Error -->
      <Message v-else-if="error" severity="warn" :closable="false">
        {{ error }}
      </Message>

      <!-- Empty -->
      <div v-else-if="releases.length === 0" class="text-center py-12 text-gray-500">
        <i class="pi pi-info-circle text-3xl mb-2" />
        <p>No hay novedades disponibles todavía.</p>
      </div>

      <!-- Releases list -->
      <Card v-for="release in releases" :key="release.id" v-else>
        <template #title>
          <div class="flex items-center justify-between">
            <span class="text-xl font-semibold">{{ release.tag_name }}</span>
            <span class="text-sm text-gray-500">{{ formatDate(release.published_at) }}</span>
          </div>
        </template>
        <template #content>
          <!-- Markdown content -->
          <div
            v-if="release.body"
            class="changelog-content text-sm text-gray-700"
            v-html="renderMarkdown(release.body)"
          />
          <p v-else class="text-sm text-gray-400 italic">Sin descripción de cambios.</p>

          <a
            :href="release.html_url"
            target="_blank"
            rel="noopener noreferrer"
            class="inline-flex items-center gap-1 mt-4 text-xs text-primary-500 hover:underline"
          >
            <i class="pi pi-external-link" />
            Ver en GitHub
          </a>
        </template>
      </Card>
    </div>
  </Container>
</template>
```

**Markdown styling note**: The `changelog-content` class needs basic styles for rendered markdown. Options:
1. Add `@tailwindcss/typography` and use `prose` class
2. Add a small scoped style block (against project convention but pragmatic for third-party HTML)
3. Use a global CSS rule for `.changelog-content h3, .changelog-content ul, ...`

Recommendation: Try `@tailwindcss/typography` first. If incompatible with Tailwind 4.1, use option 3 in the app's global styles.
