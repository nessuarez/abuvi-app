# Frontend Implementation Plan: feat-3-payment-concept-lines — Payment Concept Lines Display

## Overview

Display the structured concept lines (payment breakdown) within each payment installment card. For P1/P2, show a per-member breakdown with name, age category, attendance period, and amount. For P3, show an aggregated per-extra breakdown. This follows the existing Vue 3 Composition API patterns with PrimeVue + Tailwind CSS, extending the current `PaymentInstallmentCard` component and related types.

## Architecture Context

- **Components to modify**:
  - `frontend/src/components/payments/PaymentInstallmentCard.vue` — render concept lines inside each card
- **Components to create**:
  - `frontend/src/components/payments/PaymentConceptLines.vue` — reusable sub-component for rendering member/extra lines
- **Types to modify**:
  - `frontend/src/types/payment.ts` — add `PaymentConceptLine`, `PaymentExtraConceptLine` interfaces and extend `PaymentResponse`
- **Composables**: No changes needed — `usePayments.ts` already fetches `PaymentResponse` objects; new fields will be automatically available once types are updated
- **Routing**: No changes needed
- **State management**: No Pinia store changes — payment data remains in local composable state

### Step 1: Define TypeScript Interfaces

- **File**: `frontend/src/types/payment.ts`
- **Action**: Add concept line types and extend existing response interfaces
- **Implementation Steps**:

  1. Add `PaymentConceptLine` interface:
     ```typescript
     export interface PaymentConceptLine {
       personFullName: string
       ageCategory: string       // "Adulto" | "Niño" | "Bebé"
       attendancePeriod: string   // "Completo" | "1ª Semana" | "2ª Semana" | "Fin de semana"
       individualAmount: number
       amountInPayment: number
       percentage: number
     }
     ```

  2. Add `PaymentExtraConceptLine` interface:
     ```typescript
     export interface PaymentExtraConceptLine {
       extraName: string
       quantity: number
       unitPrice: number
       totalAmount: number
       userInput: string | null
       pricingType: string        // "PerPerson" | "PerFamily"
     }
     ```

  3. Extend `PaymentResponse` with two new optional fields:
     ```typescript
     conceptLines: PaymentConceptLine[] | null
     extraConceptLines: PaymentExtraConceptLine[] | null
     ```

  4. Extend `AdminPaymentResponse` with the same two fields (it extends `PaymentResponse`, so they should be inherited automatically — verify this).

- **Notes**: Use `string` for `ageCategory`, `attendancePeriod`, and `pricingType` rather than union types, since these are display labels generated by the backend. The frontend renders them as-is.

### Step 2: Create PaymentConceptLines Component

- **File**: `frontend/src/components/payments/PaymentConceptLines.vue`
- **Action**: Create a new component that renders concept lines as a collapsible detail section
- **Component Signature**:
  ```typescript
  defineProps<{
    conceptLines: PaymentConceptLine[] | null
    extraConceptLines: PaymentExtraConceptLine[] | null
    installmentNumber: number
  }>()
  ```

- **Implementation Steps**:

  1. **Guard**: If both `conceptLines` and `extraConceptLines` are null or empty, render nothing (early return with `v-if`).

  2. **For P1/P2 (member lines)** — when `conceptLines` has items:
     - Render a lightweight table/list inside a collapsible section (use PrimeVue `Panel` with `toggleable` or a simple Tailwind `details/summary` element for minimal weight)
     - Header label: `"Desglose del plazo"` or `"Detalle del pago"`
     - Each row shows:
       - Person name (bold): `line.personFullName`
       - Category badge: `line.ageCategory` (use subtle colored text — same style as `PaymentStatusBadge` but for age categories)
       - Period: `line.attendancePeriod`
       - Amount: `line.individualAmount` formatted as EUR with `(line.percentage%)` suffix → `line.amountInPayment` formatted as EUR
     - Footer row: **Total** = sum of all `amountInPayment` values, formatted as EUR

  3. **For P3 (extra lines)** — when `extraConceptLines` has items:
     - Same collapsible container
     - Each row shows:
       - Extra name: `line.extraName`
       - Pricing type label in parentheses: `"por persona"` / `"por familia"` (map from `line.pricingType`)
       - Quantity and unit price: `x${line.quantity} @ ${formatCurrency(line.unitPrice)}`
       - Total: `= ${formatCurrency(line.totalAmount)}`
       - If `line.userInput`: show in smaller muted text below the line
     - Footer row: **Total** = sum of all `totalAmount` values

  4. **Currency formatting**: Use the same `Intl.NumberFormat('es-ES', { style: 'currency', currency: 'EUR' })` pattern already used in `PaymentInstallmentCard.vue`.

- **Dependencies**: PrimeVue `Panel` (already in project), Tailwind utility classes
- **Implementation Notes**:
  - Keep the component lightweight — this is supplementary info, not the primary display
  - Default to collapsed state so the card doesn't become too tall
  - Use responsive layout: on mobile, stack fields vertically; on desktop, use a compact horizontal layout
  - No API calls in this component — it's purely presentational

## Error Handling Patterns

- **Null data**: Both `conceptLines` and `extraConceptLines` are nullable. The component uses `v-if` guards to render nothing when data is absent. No error state needed.
- **Malformed data**: If the backend returns unexpected shapes (shouldn't happen with typed API), the component will simply not render matching rows. No crash risk since all fields are read-only display.
- **No additional API calls**: This component is purely presentational — it consumes data already fetched by the existing `usePayments` composable. No new loading/error states required.

## UI/UX Considerations

- **Layout**: Concept lines are rendered inside the existing `PaymentInstallmentCard` as a collapsible sub-section. This preserves the current card-based layout while adding drill-down capability.
- **PrimeVue components used**:
  - `Panel` (toggleable) — for collapsible concept lines section
  - Existing components in parent: `Tag`, `Button`, `FileUpload`
- **Tailwind CSS**:
  - Use `text-sm` for concept line text (secondary info)
  - `divide-y` for row separation
  - `font-semibold` for totals row
  - `text-surface-500` for muted labels (pricing type, percentage)
  - Responsive: `flex flex-col sm:flex-row` for mobile/desktop adaptation
- **Accessibility**:
  - PrimeVue `Panel` has built-in ARIA for collapsible sections
  - Currency values include proper `aria-label` for screen readers
- **Loading states**: Not applicable — data arrives with the payment response, no separate loading.
